Conceitos Fundamentais de CSS

Como funciona a regra da cascata no CSS?

A regra da cascata no CSS define a prioridade entre regras de estilo conflitantes, usando uma combinação de origem, especificidade e ordem de declaração. Quando múltiplas regras se aplicam a um mesmo elemento, o navegador segue uma ordem de precedência, começando pelas folhas de estilo do navegador, passando pelas folhas do usuário e chegando aos estilos do autor (externos, internos e inline), determinando qual estilo será aplicado com base em sua importância e nível de detalhe no seletor.

O que é especificidade e qual a diferença entre a especificidade do seletor pela tag, seletor pela classe e seletor pelo id.

A especificidade em CSS é um critério que determina qual regra de estilo será aplicada quando há conflitos entre diferentes seletores. A diferença entre os seletores de tag, classe e ID é a sua hierarquia de especificidade, com seletores de ID sendo os mais específicos, seguidos pelas classes e, por último, as tags, que são as menos específicas. Um seletor mais específico tem maior peso e, portanto, prevalece sobre seletores menos específicos.

Explique a combinação de seletores, mostrando a combinação com espaço .primeiro h1, a combinação de filho .primeira > h1, a combinação de soma .primeira h1 + h1.

A combinação de seletores em CSS permite selecionar elementos com base em suas relações hierárquicas. A combinação com espaço, como .primeiro h1, seleciona todos os elementos h1 que estão dentro de algum descendente de .primeiro, não importa o quão profundamente aninhados estejam. A combinação de filho direto, .primeira > h1, seleciona apenas os elementos h1 que são filhos diretos de .primeira. Já a combinação de irmãos adjacentes, .primeira h1 + h1, seleciona o h1 que vem imediatamente após outro h1 dentro de .primeira.

Explique o funcionamento das pseudo-classes: first-child, last-child, nth-child, nth-of-type

As pseudo-classes em CSS permitem selecionar elementos com base em seu estado ou posição. Veja o funcionamento das mais comuns relacionadas à posição:

  • :first-child – seleciona o primeiro filho de um elemento pai.
  • Ex: p:first-child seleciona o <p> se ele for o primeiro filho do pai.
  • :last-child – seleciona o último filho de um elemento pai.
  • Ex: li:last-child seleciona o último <li> dentro de uma lista.
  • :nth-child(n) – seleciona o n-ésimo filho, onde n pode ser um número, fórmula ou palavra-chave (como odd ou even).
  • Ex: tr:nth-child(2) seleciona o segundo <tr> dentro de uma tabela.
  • :nth-of-type(n) – semelhante ao nth-child, mas considera somente elementos do mesmo tipo.
  • Ex: p:nth-of-type(2) seleciona o segundo <p> entre os irmãos <p>.

Essas pseudo-classes ajudam a aplicar estilos com base na posição dos elementos no DOM.

Explique as diferenças entre display inline e display block, inclusive as propriedades que podem ser aplicadas a um que não se aplicam ao outro.

Elementos com display: inline ocupam apenas o espaço necessário e ficam na mesma linha que outros elementos, não iniciando uma nova linha. Já elementos com display: block ocupam toda a largura disponível e sempre começam em uma nova linha.

Em termos de propriedades, elementos block aceitam largura (width), altura (height), margens verticais, e preenchimentos verticais, enquanto elementos inline geralmente não aplicam corretamente width, height, nem margens e paddings verticais.

Quais elementos são por padrão display inline e quais são display block.

Por padrão, os elementos HTML com display: block são <div>, <p>, <h1> a <h6>, e <li>, ocupando toda a largura e iniciando em uma nova linha, enquanto elementos com display: inline incluem <a>, <span>, <img>, e <strong>, que se posicionam lado a lado sem quebras de linha, mas não aceitam definições de altura e largura.

Explique o serventia das propriedades margin, padding e border, bem como TODAS as opções possíveis de shorthand (1 valor, 2 valores, 3 valores e 4 valores), os sufixos: block, inline, top, bottom, left e right.

Margin, Padding e Border são propriedades do CSS para controlar o espaço em volta e dentro de um elemento. Margin cria um espaço externo entre o elemento e os outros, enquanto Padding cria um espaço interno entre o conteúdo do elemento e a sua própria borda. A propriedade Border desenha uma linha em volta do elemento.

Shorthands:

  • 1 valor (ex: margin: 10px;): Aplica o valor aos quatro lados (top, right, bottom, left).
  • 2 valores (ex: padding: 10px 20px;): O primeiro valor para top/bottom, o segundo para left/right.
  • 3 valores (ex: border-width: 10px 20px 30px;): O primeiro para top, o segundo para left/right, o terceiro para bottom.
  • 4 valores (ex: margin: 10px 20px 30px 40px;): Na ordem: top, right, bottom, left.

Sufixos:

  • -top, -bottom, -left, -right: Especificam um lado individual (ex: margin-top: 10px;).
  • -block: Para direção do bloco (geralmente top e bottom em línguas LTR).
  • -inline: Para direção em linha (geralmente left e right em línguas LTR).

Mostre a utilização da propriedade box-sizing: border-box

A propriedade box-sizing: border-box; faz com que a largura e a altura definidas para um elemento incluam também o seu preenchimento (padding) e a sua borda, ao contrário do padrão content-box, que as soma separadamente, resultando num tamanho total maior do que o especificado. A sua utilização é recomendada para ter um controlo mais preciso sobre o layout, pois o tamanho total do elemento permanece o que foi definido.

Exemplo de uso:

.elemento {
    width: 200px;
    padding: 20px;
    border: 5px solid black;
    box-sizing: border-box; /* Com isso, a largura total será 200px */
}
